# Dummy Investor

Content, book, and course project about long-term investing. Positioned for calm, mature, anti-hype financial education — **slow wealth > fast mistakes**.

This repo contains the full operating system: strategy, agent briefs, content pipeline, book publishing plan, course design, master timeline, and weekly reviews.

## Where to start

Open [`_Project Plan/README.md`](_Project%20Plan/README.md) — it explains how the project runs, how to start a session, and what each folder does.

For current project status, open [`_Project Plan/PROJECT_STATE.md`](_Project%20Plan/PROJECT_STATE.md) — it's the single source of truth.

## Repo structure

```
├── _Project Plan/           The brain — read this first
│   ├── README.md            How the project runs
│   ├── PROJECT_STATE.md     Where we are right now
│   ├── 01_Strategy/         Brand Bible, diagnosis, metrics
│   ├── 02_Content/          Reels, hook library, topic pipeline
│   ├── 03_Book Publishing/  Tom's First Million → KDP
│   ├── 04_Course/           Course (currently paused)
│   ├── 05_Funnel & Sales/   Bio, links, landing pages, email
│   ├── 06_Weekly Reviews/   One file per week
│   ├── Agents/              Agent role briefs
│   └── _Operations/         Git, email, master timeline
├── Book/                    Book source files (PNG pages gitignored)
├── Course/                  Course recordings (videos gitignored)
├── Videos/                  Reel source files (videos gitignored)
├── Avatar/                  Brand images
├── Assets/                  Design assets
├── Music/                   Audio assets (gitignored)
└── Sound Effects/           Audio assets (gitignored)
```

Only the `_Project Plan/` folder is version-controlled in detail — everything else has large media files excluded by `.gitignore`.

## How the project is run

The project runs through **named agents**, each with a narrow scope (Content Strategist, Reel Producer, Book Publisher, Funnel Builder, Analytics, Course Architect). Each chat session is one agent doing one thing, with context files defined up front. See `_Project Plan/Agents/00 - How Agents Work.md`.
